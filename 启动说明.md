# OpenManus 服务启动说明

## 当前状态

- ✅ **前端**: 已在运行 (http://localhost:5174)
- ❌ **后端**: 需要手动启动 (http://localhost:8080)

## 手动启动后端

由于 conda 环境在当前 shell 中无法直接访问，请按以下步骤手动启动：

### 方法 1: 使用终端手动启动（推荐）

1. **打开新终端窗口**

2. **激活 conda 环境**:
   ```bash
   source ~/.zshrc  # 或 source ~/.bashrc
   conda activate openmanus
   ```

3. **进入项目目录**:
   ```bash
   cd /Users/wy770/Resume_OpenMauns/OpenManus
   ```

4. **安装依赖**（如果尚未安装）:
   ```bash
   pip install -r requirements.txt
   ```

5. **启动后端服务**:
   ```bash
   python -m uvicorn app.web.server:app --host 0.0.0.0 --port 8080 --reload
   ```

### 方法 2: 使用启动脚本

```bash
cd /Users/wy770/Resume_OpenMauns/OpenManus
# 确保已激活 openmanus 环境
conda activate openmanus
bash start_web.sh
```

## 验证服务

启动后，访问以下地址验证：

- **前端**: http://localhost:5174
- **后端 API 文档**: http://localhost:8080/docs
- **后端健康检查**: http://localhost:8080/health

## 注意事项

1. **conda 环境**: 确保 `openmanus` conda 环境已创建并包含所有依赖
2. **端口占用**: 如果端口被占用，可以修改端口或停止占用进程
3. **依赖安装**: 如果遇到模块缺失错误，运行 `pip install -r requirements.txt`

## 停止服务

- 后端: 在运行后端的终端按 `Ctrl+C`
- 前端: 在运行前端的终端按 `Ctrl+C`

## 后台运行（可选）

如果需要后台运行，可以使用 `nohup` 或 `screen`/`tmux`:

```bash
# 后端后台运行
nohup python -m uvicorn app.web.server:app --host 0.0.0.0 --port 8080 --reload > logs/backend.log 2>&1 &

# 前端后台运行
cd frontend
nohup npm run dev > ../logs/frontend.log 2>&1 &
```
