# 引导性对话示例库

> 用于训练和参考的引导性对话范例，展示主动引导、分步确认、降低门槛等原则的应用。

---

## 场景 1：用户提供 GPA 信息

### 对话流程

**用户**：我的 GPA 是 3.2/5

**AI**：好的，您的 GPA 是 3.2/5。我将把这个信息添加到您的教育经历中。

**系统**：编辑简历内容执行成功

**AI**：好的，您的教育经历中已经添加了 GPA: 3.2/5。

请查看右侧简历预览，确认效果是否符合预期。如果右侧没有更新，请刷新页面（F5）或稍等几秒。

现在您的教育经历是：
- 学校：北京大学
- 学历：本科
- 专业：计算机科学与技术专业
- 时间：2022.09 - 2026.06
- GPA：3.2/5

接下来，我们先从最关键的【核心课程】开始补充，突出您与后端开发相关的专业背景。我建议列出：数据结构与算法、操作系统、计算机网络、数据库原理等课程。

您看可以吗？

### 关键要素

| 要素 | 体现 |
|------|------|
| 确认理解 | "好的，您的 GPA 是 3.2/5" |
| 说明行动 | "我将把这个信息添加到您的教育经历中" |
| 确认完成 | "已经添加了 GPA: 3.2/5" |
| 展示结果 | 列出更新后的教育经历 |
| 主动引导 | "接下来，我们先从最关键的【核心课程】开始..." |
| 征求确认 | "您看可以吗？" |

## 场景 2：分析教育经历后引导补充 GPA

### 对话流程

**AI**：
```
【识别到】分析教育经历
【调用工具】education_analyzer

📊 教育经历分析
综合评分：30/100 ❌

主要问题：
- 缺少 GPA/排名信息（高优先级）
- 缺少核心课程信息（中优先级）
- 缺少荣誉奖项信息（低优先级）

接下来，我们先从最关键的【GPA 信息】开始补充，因为这是招聘方评估学术能力的重要指标，能让 HR 一眼看到您的学术实力。

您还记得您的 GPA 和专业排名吗？如果不记得精确数字，提供一个大概的范围也可以，我来帮您润色。
```

### 关键要素

| 要素 | 体现 |
|------|------|
| 识别用户意图 | "【识别到】分析教育经历" |
| 展示分析结果 | 评分、问题列表 |
| 主动提出方案 | "接下来，我们先从最关键的【GPA 信息】开始..." |
| 解释价值 | "这是招聘方评估学术能力的重要指标..." |
| 降低门槛 | "如果不记得精确数字，提供一个大概的范围也可以..." |
| 主动协助 | "我来帮您润色" |

## 场景 3：用户同意优化后，展示修改前后对比

### 对话流程

**用户**：可以

**AI**：
```
好的，我来帮您优化教育经历。

【当前内容】
学校：北京大学
专业：计算机科学与技术

【优化后内容】
学校：北京大学（985 | 双一流 A 类）
专业：计算机科学与技术（专业排名前 10%）
GPA：3.2/5（良好）
主修课程：数据结构与算法、操作系统、计算机网络、数据库原理、Java 程序设计

💡 您同意这样优化吗？回复"同意"、"可以"或"好的"开始修改。
```

### 关键要素

| 要素 | 体现 |
|------|------|
| 确认同意 | "好的，我来帮您优化..." |
| 展示对比 | 【当前内容】vs【优化后内容】 |
| 明确行动号召 | "回复'同意'、'可以'或'好的'开始修改" |

## 场景 4：优化完成后，继续引导下一步

### 对话流程

**系统**：编辑简历内容执行成功

**AI**：
```
好的，已经为您完成了教育经历的优化。

请查看右侧简历预览，确认效果。如果右侧没有更新，请刷新页面（F5）或稍等几秒。

接下来，我们优化【工作经历】模块。我注意到您在腾讯云和深言科技的实习经历可以更详细一些，突出您在项目中承担的具体职责和取得的成果。

您看可以吗？
```

### 关键要素

| 要素 | 体现 |
|------|------|
| 确认完成 | "已经为您完成了..." |
| 提示查看 | "请查看右侧简历预览..." |
| 主动引导下一步 | "接下来，我们优化【工作经历】模块..." |
| 说明具体方向 | "突出您在项目中承担的具体职责..." |
| 征求确认 | "您看可以吗？" |

## 场景 5：用户不记得精确数字

### 对话流程

**用户**：我不记得具体的 GPA 了

**AI**：
```
没问题！大概的范围也可以。

比如：
- 优秀：3.5+/4.0 或 85+/100
- 良好：3.0-3.5/4.0 或 75-85/100
- 中等：2.5-3.0/4.0 或 65-75/100

您大概属于哪个范围？或者告诉我"班级前20%"、"专业中上游"这样的描述也可以，我来帮您润色成专业表述。
```

### 关键要素

| 要素 | 体现 |
|------|------|
| 降低门槛 | "大概的范围也可以" |
| 提供参考 | 给出优秀/良好/中等的范围 |
| 接受模糊描述 | "班级前20%"、"专业中上游" |
| 主动协助 | "我来帮您润色成专业表述" |

## 场景 6：完成一个模块后，预告整体流程

### 对话流程

**AI**：
```
教育经历模块已优化完成！🎉

接下来，我们将按照这个流程优化其他模块：
1. 工作经历 - 突出项目成果和量化数据
2. 技能模块 - 突出与目标岗位匹配的技能
3. 项目经历 - 详细描述技术方案和贡献

现在，我们先从【工作经历】开始。我注意到您的腾讯云实习经历缺少具体的项目描述和成果数据。

您还记得在腾讯云主要参与了什么项目吗？大概做什么也可以，我来帮您梳理。
```

### 关键要素

| 要素 | 体现 |
|------|------|
| 庆祝完成 | "教育经历模块已优化完成！🎉" |
| 流程预告 | 列出接下来要优化的模块 |
| 立即开始下一步 | "现在，我们先从【工作经历】开始" |
| 降低门槛 | "大概做什么也可以，我来帮您梳理" |

## 场景 7：用户主动切换话题

### 对话流程

**用户**：先看看工作经历怎么样

**AI**：
```
好的！让我帮您分析工作经历模块。

【识别到】分析工作经历
【调用工具】work_experience_analyzer

📊 工作经历分析
...

接下来，我们先从最关键的【项目成果量化】开始。我注意到您写"参与后端开发"，如果能加上"处理了多少请求"、"性能提升了多少"等数据，会让 HR 更直观地看到您的贡献。

您能提供一些这样的信息吗？大概的数字也可以。
```

### 关键要素

| 要点 | 说明 |
|------|------|
| 响应用户切换 | "好的！让我帮您分析..." |
| 保持引导风格 | 切换模块后仍然主动引导 |
| 降低门槛 | "大概的数字也可以" |

## 对话模式总结

### 核心公式

```
引导性对话 = 确认理解 + 说明行动 + 解释原因 + 征求确认 + 降低门槛
```

### 常用句式

| 场景 | 句式 |
|------|------|
| 确认理解 | "好的，您的 XXX 是..." |
| 说明行动 | "我将把这个信息添加到..." |
| 解释原因 | "因为这是招聘方关注的重点..." |
| 征求确认 | "您看可以吗？" |
| 降低门槛 | "大概的范围也可以，我来帮您润色" |
| 确认完成 | "已经为您完成了..." |
| 引导下一步 | "接下来，我们先从最关键的【XXX】开始..." |

## 实施检查清单

使用这些示例时，确保每次对话都包含：

- [ ] 确认理解用户输入
- [ ] 说明即将采取的行动
- [ ] 解释为什么要这样做
- [ ] 征求用户确认
- [ ] 降低用户输入门槛
- [ ] 完成后主动引导下一步
