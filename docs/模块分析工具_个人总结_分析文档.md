# 个人总结分析工具设计方案

> **模块标识**：`summary`（对应简历中的 `basic.summary` 字段）
> **优先级权重**：中高（简历的"门面"，HR 首先看到的内容）
> **分析方式**：NLP 语义分析 + 模板匹配 + LLM 生成

---

## 1. 概述

### 1.1 模块定位

个人总结（Professional Summary）是简历中最精炼的部分，通常位于个人信息之后，用 2-3 句话概括候选人的核心优势和求职意向。这是 HR 阅读简历时首先看到的关键内容。

### 1.2 核心设计理念

**"30秒原则"**：
- HR 平均只用 30 秒浏览一份简历
- 个人总结决定了 HR 是否继续阅读
- 必须用最少的话传达最多的信息

**黄金公式**：
```
优秀个人总结 = 核心优势 + 经验年限 + 技术栈 + 求职意向
```

### 1.3 分析目标

| 目标 | 说明 |
|------|------|
| **完整性检查** | 确保个人总结存在且不为空 |
| **质量评估** | 评估是否包含核心要素 |
| **吸引力分析** | 评估对 HR 的吸引力 |
| **优化建议** | 提供具体的改写方案 |

## 2. 分析维度

### 2.1 完整性检查

#### 必备要素检测

| 要素 | 重要性 | 检测方法 |
|------|--------|----------|
| 经验年限 | 必须 | 数字 + "年" + "经验" |
| 核心技能 | 必须 | 技术名词、技能关键词 |
| 求职意向 | 建议 | 目标岗位、方向描述 |
| 亮点成就 | 加分 | 量化成果、特色经历 |

#### 常见缺失问题

```python
# 缺失类型识别
missing_types = {
    "empty": "完全空白",
    "too_short": "少于20字，信息不足",
    "missing_exp": "缺少经验年限",
    "missing_skills": "缺少技能描述",
    "missing_goal": "缺少求职意向"
}
```

### 2.2 质量评估

#### 优秀个人总结标准

**结构完整**：
```
✅ 好的结构：
"3年Java后端开发经验，熟悉Spring Boot、MySQL、Redis等技术栈，
有电商高并发系统开发经验，求职后端开发工程师岗位。"

❌ 差的结构：
"我是一名积极进取的程序员，热爱学习新技术。"
```

**核心要素齐全**：

| 要素 | 检测点 | 示例 |
|------|--------|------|
| 经验 | "X年经验" | "3年Java开发经验" |
| 技术栈 | 具体技术 | "熟悉Spring、MySQL" |
| 领域 | 业务领域 | "电商、金融科技" |
| 成就 | 量化成果 | "支撑X万用户" |

#### 质量评分标准

| 等级 | 分数 | 特征 |
|------|------|------|
| 优秀 | 90-100 | 结构完整，要素齐全，有亮点 |
| 良好 | 75-89 | 结构基本完整，要素较全 |
| 一般 | 60-74 | 有基本要素，但不够具体 |
| 较差 | < 60 | 要素缺失或过于空泛 |

### 2.3 吸引力分析

#### 吸引要素识别

```python
attraction_factors = {
    "大厂背景": ["腾讯", "阿里", "字节", "美团", "华为"],
    "名校背景": ["985", "211", "清北", "复交"],
    "核心技能": ["架构", "高并发", "分布式", "微服务"],
    "量化成果": ["万", "亿", "%", "倍", "TOP"],
    "特殊成就": ["开源", "专利", "论文", "获奖"]
}
```

#### 吸引力评分

```python
attraction_score = 0
for factor in attraction_factors:
    if factor in summary:
        attraction_score += factor.weight
```

### 2.4 常见问题模式

#### 问题模式识别

| 问题模式 | 示例 | 优化方向 |
|---------|------|----------|
| 空泛描述 | "热爱学习，积极进取" | 具体化技能和成果 |
| 信息过载 | 堆砌大量技术名词 | 精简为核心技能 |
| 无关信息 | "喜欢打篮球、旅游" | 删除，聚焦职业 |
| 过于谦虚 | "希望能学习更多" | 改为展示优势 |

#### 问题关键词检测

```python
problematic_keywords = {

    "irrelevant": ["喜欢", "爱好", "兴趣"],

}
```

```markdown

**问题 2：缺少核心信息**

- 未说明技术栈
- 未说明求职意向

**问题 3：无吸引力**
```

## 6. 特殊场景处理

### 6.1 不同经验水平

| 水平 | 策略 |
|------|------|
| 应届生 | 突出专业、技能、项目 |
| 1-3年 | 突出技术栈、项目经验 |
| 3-5年 | 突出业务领域、成果 |
| 5年+ | 突出架构能力、管理经验 |

### 6.2 转岗场景

```
转岗个人总结要点：
1. 说明原领域经验年限
2. 强调可迁移技能
3. 说明转岗原因和学习成果
4. 表达对新领域的热情
```

### 6.3 跨国/外企简历

```
外企简历个人总结特点：
1. 使用英文
2. 更强调成就和影响力
3. 通常以 "Highly motivated..." 开头
4. 强调软技能和领导力
```
