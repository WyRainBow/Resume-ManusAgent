# 开源经历分析工具设计方案

> **模块标识**：`opensource`（对应简历中的 `openSource` 字段）
> **优先级权重**：中（有则加分明，无则不影响）
> **分析方式**：GitHub API + 项目分析 + LLM 评估

---

## 1. 概述

### 1.1 模块定位

开源经历是简历中的**加分项**，特别是对于技术岗位。优秀的开源经历可以证明候选人的：
- 技术能力（代码质量）
- 学习能力（接触新技术）
- 协作能力（团队合作）
- 影响力（Star、Fork、贡献度）

### 1.2 核心设计理念

**影响力导向**：
- 开源经历的价值在于**影响力**而非数量
- 重点评估：Star 数、贡献度、项目活跃度
- 区分：核心贡献者 vs 普通参与者

**质量 > 数量**：
- 1 个高质量项目 > 10 个低质量项目
- 核心 PR > 简单的 typo 修复
- 长期维护 > 一次性贡献

### 1.3 分析目标

| 目标 | 说明 |
|------|------|
| **真实性验证** | 验证 GitHub 仓库是否存在 |
| **影响力评估** | 评估 Star、Fork、贡献量 |
| **技术关联** | 分析与目标岗位的技术关联度 |
| **贡献分析** | 评估贡献的质量和价值 |

## 2. 分析维度

### 2.1 项目基本信息

#### 项目识别

| 信息 | 获取方式 | 用途 |
|------|----------|------|
| 仓库名称 | 用户输入/GitHub API | 验证存在性 |
| 项目描述 | GitHub API | 了解项目内容 |
| 主要语言 | GitHub API | 技术栈匹配 |
| 创建时间 | GitHub API | 项目持续性 |
| 更新时间 | GitHub API | 项目活跃度 |

#### 项目分类

```python

```

### 2.2 影响力评估

#### 量化指标

| 指标 | 权重 | 评分标准 |
|------|------|----------|
| Star 数 | 30分 | 1000+=30, 100+=20, 10+=10, <10=5 |
| Fork 数 | 15分 | 100+=15, 10+=10, 1+=5 |
| Watch 数 | 10分 | 关注度指标 |
| 贡献数 | 30分 | PR数量 x 质量系数 |
| 活跃度 | 15分 | 最近更新时间 |

#### 影响力等级

| 等级 | Star范围 | 评价 |
|------|----------|------|
| 爆款 | > 10000 | 🔥 极具影响力 |
| 热门 | 1000 - 10000 | ⭐ 高影响力 |
| 受欢迎 | 100 - 1000 | ✅ 良好影响力 |
| 一般 | 10 - 100 | 📌 初步认可 |
| 较少 | < 10 | 📝 个人练习 |

## 4. 匹配度评分规则

### 4.1 评分标准

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 影响力 | 40分 | Star数权重 |
| 贡献质量 | 35分 | 贡献类型和深度 |
| 技术匹配 | 25分 | 技术栈关联度 |

### 4.2 优先级计算

```python
priority_score = (100 - score) + severity_bonus

# 开源经历优先级通常较低（因为不是必须的）
high   = +15  # 无开源经历且技术岗位
medium = +5   # 开源经历质量低
low    = +0   # 无问题

# 例子：开源经历 score=70，无重大问题
priority_score = (100 - 70) + 0 = 30
```

## 6. 特殊场景处理

### 6.1 无开源经历

| 场景 | 处理方式 |
|------|----------|
| 无任何开源 | 不扣分，建议补充项目经历 |
| 只有 Fork | 按项目经历分析 |
| 有个人项目未发布 | 建议发布到 GitHub |

### 6.2 不同经验水平

| 水平 | 开源期望 |
|------|----------|
| 应届生 | 有个人项目即可 |
| 1-3年 | 有个人项目或贡献记录 |
| 3-5年 | 有一定影响力的项目 |
| 5年+ | 最好有核心贡献 |

### 6.3 非技术岗位

| 岗位 | 开源经历价值 |
|------|-------------|
| 研发 | 高价值加分项 |
| 产品 | 中等价值（侧重产品设计类） |
| 运营 | 低价值（非必须） |
